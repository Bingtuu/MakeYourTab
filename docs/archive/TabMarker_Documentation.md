## 2. 产品概述

### 2.1 用户场景

在日常工作和学习中，用户经常需要同时打开多个浏览器标签页。当标签页数量过多时，寻找特定页面变得困难，需要一个一个点击查看。TabMarker插件解决这一痛点，允许用户通过颜色、emoji符号、文字tag等方式对标签页进行标记，实现快速定位。

**典型使用场景：**
- 开发人员同时查阅多个技术文档和API参考页面
- 学术研究人员浏览多篇论文和参考资料
- 运营人员管理多个数据分析和报表页面
- 普通用户浏览新闻、购物、娱乐等多类型网页

### 2.2 产品目标

TabMarker的核心目标是让浏览器标签页管理变得高效、直观、个性化。

| 目标 | 说明 |
|------|------|
| 高效 | 通过颜色、emoji和文字快速定位目标标签页 |
| 直观 | 标记可视化，一眼识别标签页类型 |
| 个性 | 支持用户自定义标记组合 |
| 本地 | 所有数据仅存储在本地，保护隐私 |

# 二、技术规范文档

## 1. 技术架构

### 1.1 整体架构

TabMarker采用Chrome Extension Manifest V3架构，主要由以下组件构成：
- 后台脚本 (Background Script / Service Worker)
- 弹窗页面 (Popup)
- 内容脚本 (Content Script)
- 选项页面 (Options Page)

### 1.2 组件说明

#### 1. 后台脚本 (Background Script)

后台脚本作为服务工作运行，负责处理插件的核心业务逻辑。
- 监听快捷键事件
- 管理标签页标记状态
- 与弹窗页面通信

#### 2. 弹窗页面 (Popup)

弹窗页面是用户主要的交互入口，提供标记设置界面。
- 显示当前标签页的标记状态
- 提供颜色、emoji、文字标记设置
- 展示常用tag组合

#### 3. 内容脚本 (Content Script)

内容脚本注入到网页中，用于处理页面级的交互。
- 监听页面事件
- 与后台脚本通信

#### 4. 选项页面 (Options Page)

选项页面提供插件的详细设置界面。
- 快捷键管理
- 常用tag组合管理
- 插件设置

# 三、隐私与安全规范

## 1. 隐私政策

### 1.1 核心原则

TabMarker插件严格遵守"本地化"原则，所有用户数据仅存储在用户本地浏览器，不会上传到任何外部服务器。

> **关键承诺**
> - 所有数据仅存储在本地浏览器
> - 不会向任何外部服务器上传数据
> - 不会收集用户浏览历史或个人信息
> - 不会追踪用户行为或使用情况

### 1.2 数据收集说明

插件仅收集以下必要数据：

| 数据类型 | 收集方式 | 用途 |
|----------|----------|------|
| 常用Tag组合 | 用户主动输入 | 保存常用标记组合 |
| 快捷键设置 | 用户主动设置 | 存储用户定义快捷键 |
| 标签页标记 | 用户主动设置 | 标记当前标签页 |

### 1.3 数据存储位置

所有用户数据均存储在本地，具体位置如下：
- 常用Tag组合: chrome.storage.local
- 快捷键设置: chrome.storage.local
- 标签页标记: chrome.tabs API (标签页实例)

---

## 2. 权限说明

### 2.1 所需权限

TabMarker插件需要以下权限才能正常工作：

| 权限名称 | 用途说明 |
|----------|----------|
| tabs | 读取和修改标签页标题和标记 |
| storage | 本地存储用户设置 |
| activeTab | 获取当前活动标签页信息 |
| commands | 注册键盘快捷键 |

### 2.2 权限使用说明

**tabs权限**
用于读取和修改标签页的标题和标记信息。插件仅修改标签页的标题属性，不会访问页面内容。

**storage权限**
用于在本地存储用户的常用tag组合和快捷键设置。数据仅保存在用户本地浏览器。

**activeTab权限**
用于获取当前活动标签页的信息，以便快速标记。

**commands权限**
用于注册和监听键盘快捷键，提供快速操作入口。

## 3. 安全规范

### 3.1 代码安全

**输入验证**
- 所有用户输入均需进行长度和格式验证
- 文字标签长度限制20个字符
- 禁止输入HTML和JavaScript代码
- 对特殊字符进行转义处理

**数据安全**
- 使用JSON格式存储数据
- 对存储数据进行类型检查
- 防止数据滥用，限制存储数量

### 3.2 错误处理

- 所有异步操作均需添加try-catch处理
- 错误信息不暴露敏感内容
- 提供友好的错误提示信息
